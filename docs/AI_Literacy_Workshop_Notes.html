AI Tool Recommendations

🟣 Claude (Anthropic) $20/month or $100/month

Highly Recommended! The dominant AI platform for professional use.

  • More accurate and less hallucination than ChatGPT
  • Better organization and academic-quality output
  • Superior artifact creation (files, code, diagrams)
  • $100 version: Much more memory, longer conversations

🟢 ChatGPT (OpenAI) $20/month

Good alternative with image generation capabilities.

  • Can generate images (DALL-E integration)
  • More "people-pleasing" — may have confirmation bias
  • Can upload more files at once (20+ vs 7-8)

🔵 Copilot (Microsoft) Free with university

Available but limited capabilities.

  • Sometimes uses ChatGPT backend
  • Doesn't maintain conversation memory well
  • "Even when they want to say something is bad, they say it's Copilot"

🔴 Grok (xAI) Paid

Known for brutally honest feedback.

  • Trained on Twitter/X data — reflects harsher, more direct tone
  • "It will say your paper is trash" — not people-pleasing
  • Good for getting unfiltered criticism on your work
  • Provides current sentiment data from social media

🟡 Gemini (Google) Free / Paid

Google's AI with unique integrations.

  • Direct access to Google Scholar — can find real papers
  • Notebook LM — useful for literature reviews
  • Must prompt specifically: "go to Google Scholar, do not hallucinate"
  • Presenter noted: "I personally have not used Gemini" extensively
🚨 NEVER Use Free Versions for Sensitive Data!
  • Free versions may use your data for training
  • Student names, grades, research data = HIGH RISK
  • Even "deleted" data may be retained on servers
  • Paid versions have opt-out options, but still be cautious

3. Claude Features: Chat vs Cowork

Claude has two main modes (requires downloading the desktop app):

⚠️ Windows Installation Note:

Windows users may need to enable Developer Mode to install the Claude desktop app. You'll see a message: "Trusted app installs must be enabled to install Claude."

  • Go to Settings → System → Turn on Developer Mode
  • Warning: "Apps from outside Microsoft Store could expose you to risks"
  • Mac users did not encounter this issue
  • Check your school/IT policy before enabling

💬 Chat Mode

  • Upload files directly to conversation
  • Has web search/research capabilities
  • Each chat is separate (no cross-chat memory)
  • Can activate "Research" skill for deep paper searches
  • Better for online research tasks

📁 Cowork Mode

  • Point to a folder on your computer
  • No need to upload — reads files as needed
  • Consumes LESS memory/tokens
  • Can manipulate files in place (rename, edit)
  • Better for working with many local files

Cowork Use Case Example

📂 File Renaming Example:

"I had a folder of research papers with random names. I pointed Cowork to that folder and asked it to rename all files using: Author Name - Journal Name - Year."

It went inside each PDF, extracted the information, and renamed them in place!

⚠️ Privacy Warning for Cowork:
  • Be careful which folders you give access to
  • Never point to folders containing student data
  • Use separate folders for course materials only

Projects Feature

Both Claude and ChatGPT support Projects — organized collections of chats.

  • Keep related chats together (e.g., "Python Course Materials")
  • Easy to find and continue previous work
  • Same session can be used to build on previous summaries
  • Connect new transcripts to previous sessions for continuity

4. Recording & Transcription Setup

The Dual-Screen Method

1 Use TWO devices: Personal laptop + School computer
2 Join Microsoft Teams meeting from school computer (for recording)
3 Share screen from personal laptop (for presentation)
4 Turn OFF speaker on laptop (prevents echo)
5 School computer microphone captures everything (stronger mic)
💡 Why This Works:
  • Gets both video/audio AND transcript
  • School microphone is stronger — captures student questions too
  • If students speak low, repeat their question for the recording

Microsoft Teams Transcription Quality

  • Teams transcription is extremely good
  • Even detects when you're writing code on the board
  • Separates board writing from spoken content
  • Can correct mistakes and detect context

Downloading the Transcript

  1. During meeting: Click "Show transcript" to verify it's working
  2. After meeting: Download as separate file (.docx or .vtt)
  3. Feed transcript to Claude for summarization

5. Creating HTML Course Summaries

✅ This is the MOST EFFECTIVE use of AI for teaching!

Students love having summarized notes. Even if they miss a session, they can catch up easily.

Why HTML Instead of PDF/Word?

FormatAccessibility ScoreMemory UsageEditing
HTML96-97% (excellent)SmallestEasy in Canvas
WordMediumMediumRequires download
PDFLow (disaster!)LargestVery difficult
🏛️ Texas Accessibility Requirements:

HTML automatically meets most accessibility requirements. Tags are easy to add. Colors can be adjusted. PDF/Word files often fail compliance checks.

Need help? Contact CDE (Center for Distance Education) and OIT (Office of Information Technology) — they can assist with making content accessible.

The Summarization Workflow

1 Upload transcript to Claude
2 Ask for structured HTML summary with tables, diagrams, key points
3 Request: "Integrate all CSS styles inside HTML tags" (important for Canvas!)
4 Download HTML file
5 Upload to Canvas or paste into Page HTML editor

Critical: CSS Integration

⚠️ Canvas Removes External CSS!

When creating HTML for Canvas Pages, always tell Claude:

"I want to upload this to Canvas. Integrate all CSS styles inside the HTML tags rather than a separate CSS file."

Otherwise, your page will appear as plain text with no formatting!

Maintaining Continuity

Keep the same chat session for all your course summaries:

  • Upload new transcript to the same chat
  • Say: "Continue from the previous session, enhance what we had, and provide a new summary"
  • AI will connect topics and show progression between sessions

6. Canvas Integration

Uploading HTML Files

Simply upload the HTML file to Canvas Files → Link in Modules. Students click and see the formatted page.

Creating Custom Pages

1 Go to Pages → View All Pages → Add Page
2 Switch to HTML Editor view
3 Paste your HTML code (with integrated CSS)
4 Save the page
5 Use "Set as Front Page" for welcome pages

Front Page Feature

  • Go to Pages → View All Pages
  • Select any page → "Set as Front Page"
  • Students see this when they open the course
  • Great for welcome messages, exam reminders, announcements
  • Can change it anytime (e.g., "Good luck on the exam!")

Editing After Upload

You can edit pages in the visual editor (not just HTML):

  • Click Edit on any page
  • Make changes in the WYSIWYG editor
  • No need to touch the code

7. Generating Assignments & Exams

Using Your Summaries to Create Questions

Once you have HTML summaries of your course:

  • Upload summaries to Claude Chat OR point Cowork to your folder
  • Ask for questions based on specific sessions
  • Specify: difficulty level, question types, computational vs conceptual
  • Request tables, images, or scenarios as needed

Canvas Quiz Import & Lockdown Browser

💡 Presenter's Workflow:

"I generally use the Lockdown Browser to conduct exams. After I develop the questions, I provide them to Claude and ask for the import package to Canvas."

✅ Huge Time Saver!

Ask Claude to convert your questions into a Canvas import package. Upload one file and all questions appear in your quiz — no manual entry!

💡 Quality Control Prompt:

"Make sure all questions are correct, all options make sense, there is no repetition, and convert to Canvas import format."

Question Verification

Always ask AI to verify before finalizing:

  • Check that answers are correct
  • Ensure options are plausible but distinct
  • Remove any duplicate questions
  • Verify difficulty matches your requirements

8. AI in Practice: Case Studies & Games

Creating Case Studies with Built-in Errors

🎯 Evaluation > Generation:

Instead of asking students to write (which they can outsource to AI), give them AI-generated content with deliberate problems to find!

The "20 Problems" Case Study

  1. Warm up the AI: Feed it your PowerPoint/course materials
  2. Discuss each topic: Let AI understand what you're covering
  3. Request the case: "Create a compelling case study covering these topics with 20 hidden problems"
  4. Refine: Ask for storytelling elements, game-like structure
  5. Use in class: Groups find and discuss the problems (5-10 min activity)
The first draft is usually boring. Ask AI to make it "more story-based with a gaming structure" for better engagement!

Jeopardy-Style Games

🎮 AI-Generated Classroom Games

Takes about 5 minutes to generate!

  1. Provide your case study or content
  2. Ask: "Create a Jeopardy-style game for [X] groups based on this content"
  3. Specify: round-robin format, point values, categories
  4. Claude generates interactive HTML with clickable categories
  5. Open in browser and play in class

Questions can reference specific parts of the case study — students must identify if something is a problem or not.

9. AI Failures: Cautionary Tales

These real-world examples show why AI implementation requires careful planning:

✈️ Air Canada Chatbot Lawsuit

A customer traveling for a funeral asked the chatbot about bereavement discounts. The AI said: "Buy the ticket, call us after, and we'll refund you."

Problem: No such policy existed — the AI made it up.

Result: Customer sued. Air Canada paid $800. Court ruled: Anything on your website (chatbot, AI, etc.) represents your company.

Impact: Many law firms immediately removed their chatbots.

🍔 McDonald's Voice AI Disaster

McDonald's partnered with IBM for AI-powered drive-through ordering.

Lab Testing: Perfect performance

Reality: Drive-throughs have rain, dogs, babies, unclear audio

Results:

  • Customer ordered ice cream cone → Got ice cream with bacon
  • Customer ordered diet coke → Got sweet teas
  • One customer received 200 sandwiches

Videos went viral on TikTok → Major reputation damage → McDonald's ended IBM partnership

📱 Samsung AI Data Leak

Samsung banned AI in workplace → Employee pressure → Opened access for 3 weeks

Three incidents in three weeks:

  • Highly confidential semiconductor code uploaded to ChatGPT
  • Confidential meeting information copied into AI
  • Sensitive data potentially used for training

They could not remove the data from ChatGPT.

💳 Klarna Workforce Reduction

Buy-now-pay-later company reduced workforce by ~40% (from ~5,000 to ~3,000 employees) through AI adoption, natural attrition, and a hiring freeze. Their AI chatbot handled work equivalent to 700 employees.

First 2 years: Cost savings, faster responses

After: Customer dissatisfaction increased — these businesses have emotional customer interactions (payment problems, stress). Chatbots couldn't handle the nuance.

CEO publicly admitted "we went too far" and Klarna is now rehiring human staff.

📊 Boston Consulting Group Recommendation:
  • 10% of AI budget → Algorithm
  • 20% → Technology
  • 70% → Training People

Without proper training, most AI implementations fail!

10. AI for Research

Finding Papers with Claude

1 In Claude chat, click the + button
2 Activate the Research skill
3 Ask: "Find 50 papers published in FT50 journals about [topic]"
4 Wait 20-30 minutes for deep search
5 Get properly cited, real papers (no hallucination)
📚 What is FT50?

Financial Times Top 50 — a list of the 50 most prestigious business and management journals used for faculty research rankings. Specifying "FT50 journals" ensures Claude searches for high-quality, peer-reviewed academic sources.

Paper Summarization

Upload a paper and ask Claude to summarize:

  • Introduction summary
  • Research questions
  • Literature review with references
  • Dataset used
  • Methodology
  • Implications
Claude can convert PDF summaries to Word files — Word files are smaller and easier to add to future chats!

Finding Missing References

Put bracketed sections in your draft:

"For the [square bracket part], I need more references related to this topic."

Claude will find relevant papers for that specific section.

Creating Diagrams

For research frameworks and paper figures:

  • Explain your framework to Claude
  • Ask for an SVG diagram
  • SVG format is professional and publication-ready
  • Best diagram generation compared to other tools
⚠️ Privacy Caution:

Uploading unpublished research has risks. On paid versions, you can opt out of training, but data may still be retained on servers. Consider the sensitivity of your work.

12. AI Policy & Academic Integrity

Setting Up AI Policy

✅ Effective Approach:

Put your AI policy in a Discussion Board and require students to sign/acknowledge. This creates a clear record.

Example policy elements:

  • Specify which assignments allow/prohibit AI
  • Require disclosure of AI use
  • State consequences for violations
  • Have students digitally sign acknowledgment

Detecting AI-Generated Work

Detection Methods

  • Reverse engineer the prompt: Give their work to AI and ask "How would you generate this?"
  • Quality mismatch: Sudden jump in writing quality from previous work
  • Too clean: Unusually polished for student level
  • Sophisticated functions: Code uses methods beyond course content
  • Ask questions: If they can't explain their work, they didn't do it

In-Class Verification

  • Give AI-generated content with errors
  • Students must find and fix problems
  • Evaluation happens in class (no take-home)
  • Group discussions reveal understanding
⚠️ The Student Reality:

"When you ask if they have access to AI, they say no. But when they deliver work, they clearly had access."

Education systems haven't figured out how to handle this yet.

13. The Future of Education with AI

Predicted Model: Flipped Classroom 2.0

The New Structure

  • Before Class: Videos, materials, chatbots for Q&A — students study independently
  • In Class: Activities, evaluations, presentations, skill-building
  • Assessment: Happens in-person, not take-home

"Harvard is already doing this — all materials available beforehand, class time for applications."

Future: Screen Monitoring for Online Assessments

🔮 Coming Soon:

Tools that will capture and monitor screen activity over time for online assessments:

  • Log every click, copy/paste, reading time, writing patterns
  • Predict whether AI was used based on activity patterns
  • Create platforms where students must spend real time engaging with content

Caveat: "If someone wants to cheat, they're going to figure out how to do it — you just have to increase the cost of cheating."

The Competitive Reality

🚨 Critical Point:

"We are reaching the point that if you are not using AI, you are losing competitive advantage — both in research and teaching."

Expectations are changing. Eventually, AI access will be assumed like internet access.
