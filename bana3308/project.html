1.2 Data Quality Assessment

Real-world data is never perfect. The combined class dataset will contain issues. Look for the following specific problems:

  • Blank or empty drawings: Samples where someone clicked "Add to Dataset" without drawing anything, or barely drew at all. Detect these by calculating Total_Ink (see 1.3) — if the sum of all 784 pixel values is close to zero, the sample is essentially blank and should be removed.
  • Overexposed samples: The opposite problem — samples where someone filled the entire canvas white. These have very high Total_Ink values but contain no useful letter shape. Check for outliers at the high end of Total_Ink.
  • Unbalanced contributions: Check if any group submitted far fewer samples than expected. Count the number of samples per group and per letter. Are there groups with only a handful of samples for a given letter?

For each issue you find, document what you found and what you decided to do about it (remove the sample, keep it, etc.). If you find no issues in a category, state that explicitly.

1.3 Feature Engineering

The raw pixel values (p0–p783) are the low-level representation. Use the Formula widget in Orange to create the following higher-level features:

  • Total_Ink: Add up all 784 pixel values for a single sample. This tells you how much "ink" was used. Letters with more strokes (like "E" or "B") typically have higher Total_Ink than simpler letters (like "C"). Formula: Total_Ink = p0 + p1 + p2 + … + p783.
  • Brightness: The average pixel value. Formula: Brightness = Total_Ink / 784. This normalizes Total_Ink to a 0–255 scale.

The next two features measure where the "center" of the drawn letter sits on the canvas. Imagine placing the 28×28 grid on a table and putting a small weight on every bright pixel — the heavier the weight, the brighter the pixel. The center of mass is the balancing point of all those weights.

As shown in the diagram above:

  • Letter "C" has most of its bright pixels on the left side (columns 1–2). Its center of mass is pulled to the left, giving a low Center_Mass_X value (around 2.7 out of 6).
  • Letter "D" has bright pixels on both the left (column 1) and right (column 5). Its center of mass sits closer to the middle, giving a higher Center_Mass_X value (around 3.5 out of 6).
  • Both letters have pixels evenly spread top-to-bottom, so their Center_Mass_Y values are both near the middle.
  • Center_Mass_X (horizontal center): For each pixel, multiply its brightness by its column number (0–27). Sum all products. Divide by Total_Ink. Result: average horizontal position of the ink, from 0 (far left) to 27 (far right).
  • Center_Mass_Y (vertical center): Same idea with row numbers (0–27 from top to bottom). Multiply each pixel's brightness by its row number, sum, divide by Total_Ink.
How to get each pixel's column and row number: The pixels p0–p783 are stored row by row. Since each row has 28 pixels, you can compute any pixel's position from its index: column number = index mod 28 (the remainder when dividing by 28) and row number = floor(index ÷ 28) (integer division by 28). For example, p0 is at row 0 col 0; p27 is at row 0 col 27; p28 is at row 1 col 0; p55 is at row 1 col 27; and so on. In the Formula widget, you can use p0*0 + p1*1 + p2*2 + … + p27*27 + p28*0 + p29*1 + … for Center_Mass_X, where the multiplier is the column number repeating 0–27 for each row.
Hint: Use the Formula widget in Orange for all four features.

2.3 Dimensionality Reduction with PCA

With 784 pixel features, direct visualization and modeling is challenging. Use PCA to reduce the feature space while keeping most of the information.

  • Apply PCA: Use the PCA widget in Orange on the 784 pixel features (p0–p783)
  • Variance analysis: How many components are needed to capture 90% of the total variance? Report the cumulative explained variance.
  • 2D Scatter Plot: Plot PC1 vs. PC2, colored by letter label. Do the five letters form distinct clusters? When two letters' data points overlap heavily, your models will likely struggle to tell them apart.
  • Component interpretation: Examine the loadings of PC1 and PC2. Which pixel regions contribute most?
  • Prepare PCA features: Select the number of components that capture at least 90% variance for use in Part 3.
