Element-wise vs Matrix Multiplication

Critical Distinction

A * B — Element-wise multiplication. Multiplies corresponding elements at same positions.

A @ B — Matrix multiplication. True linear algebra dot product.

Three Ways to Matrix Multiply

SyntaxStyleNotes
C = A @ BOperatorMost Pythonic, recommended
C = A.dot(B)MethodObject-oriented style
C = np.dot(A, B)FunctionNumPy function style

All three produce the same result.

The Shape Rule: Can We Multiply?

The Inner Dimensions Must Match

A has shape (m, n) · B has shape (n, p) · Result has shape (m, p)

How Matrix Multiplication Works

The Pattern

Each element in the result = Row of A · Column of B (dot product).

Position [i, j] = Sum of (Row i of A) × (Column j of B)

Order Matters: A @ B ≠ B @ A

Exam-Style Problem

You might see: "Calculate AT @ B - A"

Steps: (1) Transpose A first, (2) Matrix multiply AT @ B — check shapes are compatible, (3) Subtract A from the result — check shapes match.

Always check shape compatibility at each step!

Key Takeaways

!A * B = element-wise · A @ B = matrix multiplication.
✓Use @ operator for matrix multiplication (or .dot() or np.dot()).
✓Shape rule: (m, n) @ (n, p) → (m, p) — inner dimensions must match.
✓Calculation: Result[i, j] = Row i of A · Column j of B.
!Order matters: A @ B ≠ B @ A (not commutative!).

Code Cheat Sheet

# Matrix Multiplication (all equivalent)
C = A @ B              # @ operator (recommended)
C = A.dot(B)           # Method syntax
C = np.dot(A, B)       # Function syntax

# Element-wise multiplication (NOT matrix mult)
C = A * B              # Element by element

# Check shapes before multiplying
print(A.shape)         # (m, n)
print(B.shape)         # (n, p) — n must match!
                       # Result will be (m, p)

# Combined operations
result = A.T @ B - A   # Transpose, multiply, subtract
