1.4 — Boolean Filtering

Boolean filtering lets you select elements from an array based on conditions. It's one of the most powerful and frequently used features in NumPy.

The Boolean Mask Concept

When you apply a comparison to an array, you create a boolean array — an array of True and False values. This acts like a mask that you place on top of your data to filter out what you need.

Step 2: Using the Mask to Filter

Boolean Indexing

Put the boolean expression inside the brackets to extract only the values where the mask is True:

a[a > 5]  →  Returns: [8, 9, 6]

Important: The result is always Rank 1 (a flat 1D array), regardless of the original array's shape.

Step 3: Modifying Filtered Elements

Real-World Example: Targeted Updates

Want to give a 5% raise only to employees earning less than $20,000? Boolean filtering lets you target specific values.

a = np.array([[1, 5, 8, 9],
              [2, 3, 4, 6]])

# Add 2 only to values <= 4
a[a <= 4] += 2

Don't Forget the Parentheses!

a[a > 2 & a < 8]  →  ERROR!
a[(a > 2) & (a < 8)]  →  Correct!

Parentheses are required because of operator precedence. NumPy will throw an error without them.

1.5 — Operations Between Arrays

What happens when you add, subtract, or multiply two arrays together? The answer depends on their shapes.

The Simple Rule

Arrays need to have the exact same shape for element-wise operations. When shapes match, operations happen element by element at the exact same positions.

Exam-Style Problem: Walkthrough

Problem: Calculate B - 2*A and find the sum of all elements

Given A:

[[1, 2],
 [3, 4]]

Given B:

[[1, 3],
 [4, 5]]

Step 1: Calculate 2*A

2 * [[1, 2], [3, 4]] = [[2, 4], [6, 8]]

Step 2: Calculate B - 2*A

[[1-2, 3-4], [4-6, 5-8]] = [[-1, -1], [-2, -3]]

Step 3: Sum all elements

-1 + (-1) + (-2) + (-3) = -7

What If Shapes Don't Match?

If the shapes don't match and there's no valid broadcasting rule (covered later), NumPy will raise a ValueError:

ValueError: operands could not be broadcast together with shapes (2,3) (2,4)

Key Takeaways

✓Boolean comparisons create True/False arrays that act as masks.
✓Boolean indexing extracts values where mask is True: a[a > 5]
✓Filtered results are always 1D (flat arrays), regardless of original shape.
✓Modify in place: a[a <= 4] += 2 updates only matching elements.
!Use & and | (not and/or) with parentheses around each condition!
✓Array operations: Same shape required; operations happen position-by-position.

Code Cheat Sheet

# Boolean Filtering
a > 5                    # Creates boolean mask
a[a > 5]                 # Extract values where True
a[a <= 4] += 2           # Modify filtered elements

# Multiple Conditions
a[(a > 2) & (a < 8)]     # AND condition
a[(a < 2) | (a > 8)]     # OR condition

# Array Operations (same shape required)
A + B                    # Element-wise addition
A - B                    # Element-wise subtraction
A * B                    # Element-wise multiplication
B - 2*A                  # Combined operations
