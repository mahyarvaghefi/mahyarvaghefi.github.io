Question 1 — Linear Regression (Housing Dataset)

  • Part 1: Load the Housing dataset via the Datasets widget. Connect it to Data Table, Feature Statistics, Distributions, and Correlations to explore the data before building any model.
  • Part 2: Use Select Columns to keep features CRIM, RM, AGE, DIS, TAX, PTRATIO, LSTAT and target MEDV. Add Continuize with Z-score normalization. Add a Scatter Plot connected to the non-normalized output to explore RM vs. MEDV and LSTAT vs. MEDV.
  • Part 3: Connect Data Sampler after Continuize. Set it to 75% training / 25% test and enable Replicable (deterministic) sampling. Connect the two outputs to separate Data Table widgets labeled "Training Data" and "Test Data".
  • Part 4: Build two models — a plain Linear Regression and a Ridge Regression (set regularization to Ridge L2, slider ~3/4 of the way). Each model needs its own Predictions widget receiving Training Data as the model and Test Data as input. Also connect each regression widget to a Data Table to inspect coefficients.
  • Part 5: Connect your Predictions widget to a Scatter Plot to visualize actual vs. predicted values. Create a second Predictions widget for Model A using Training Data on both inputs (to compare in-sample vs. out-of-sample performance).

Question 2 — Classification (Breast Cancer Wisconsin)

  • Part 1: Load Breast Cancer Wisconsin via the Datasets widget. Connect to Data Table, Distributions, and Feature Statistics. Use Distributions to examine the target variable type and individual features, coloring by class.
  • Part 2: Use Select Columns to select all 9 features (Clump_thickness, Unif_Cell_Size, Unif_Cell_Shape, Marginal_Adhesion, Single_Cell_Size, Bare_Nuclei, Bland_Chromatine, Normal_Nucleoli, Mitoses) and target type. Add Continuize with Z-score normalization. Configure Data Sampler to 70% training / 30% test with Replicable and Stratified sampling enabled.
  • Part 3: Build a Logistic Regression model (regularization: None). Connect Training Data → Logistic Regression → Predictions → Confusion Matrix. Connect Test Data to the Predictions widget. Record all metrics from the Predictions widget and TP/TN/FP/FN from the Confusion Matrix.
  • Part 4: Connect your Logistic Regression widget to a Data Table to view coefficients. Also build a second Logistic Regression trained on non-normalized data (bypassing Continuize) to compare coefficient tables. Create a second Predictions widget using Training Data on both inputs to check generalization.
  • Part 5: Answer critical analysis questions based on all results gathered in Parts 1–4. No new Orange widgets needed for this part.
⚠️

Predictions widget setup: Each model requires its own Predictions widget. The model (e.g., Linear Regression widget) connects to the Predictions widget as the learner, and the Test Data connects as the data to be evaluated. These are two separate input connections on the same Predictions widget.

6 Common Mistakes to Avoid

  • Wrong dataset selected: For Question 2, search for Breast Cancer Wisconsin exactly. Selecting the wrong "Breast Cancer" dataset will give you different features and incorrect results.
  • Predictions widget wired incorrectly: The model widget (e.g., Linear Regression) connects as the learner input and the data (Test Data) connects as the data input. Both must be connected for the widget to show results.
  • Using the same Predictions widget for training and test: Parts 4 and 5 require separate Predictions widgets — one evaluating Test Data (out-of-sample) and one evaluating Training Data (in-sample). Do not reuse the same widget.
  • Normalization applied to scatter plots: When creating scatter plots in Part 2, connect to the non-normalized output of Select Columns, not the output of Continuize. Normalized values make scatter plots harder to interpret.
  • Coefficients without normalization: When comparing feature importance using coefficients, you must use the normalized version of your model. Raw coefficients from non-normalized data cannot be directly compared across features with different scales.
  • Missing explanations: Many questions ask you to explain or interpret a result — not just report a number. A number alone without an explanation will lose points.
  • Leaving placeholder text: Before submitting, search your notebook for [Your answer] to make sure no cells were missed.
