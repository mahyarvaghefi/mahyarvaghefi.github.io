1.7 — Aggregation Functions

Sometimes you don't want to keep all your numbers — you want to summarize them. That's aggregation: reducing many values to one (or a few).

Common Aggregation Functions

FunctionMethod SyntaxFunction SyntaxDescription
Suma.sum()np.sum(a)Add all elements
Meana.mean()np.mean(a)Average of all elements
Maxa.max()np.max(a)Maximum value
Mina.min()np.min(a)Minimum value
Std Deva.std()np.std(a)Standard deviation

Do NOT Use Python's Built-in sum()

sum(a) — Python's built-in (slow, unexpected results)
np.sum(a) or a.sum() — NumPy's version (fast, correct)

Always use the NumPy version for arrays!

The axis Parameter: Row vs Column Operations

How to Remember Axis

axis=0: Operation goes DOWN (column-wise) — collapses rows
axis=1: Operation goes ACROSS (row-wise) — collapses columns
No axis: Operation on ALL elements

a.sum(axis=0)   # [4, 2, 11]

a.sum(axis=1)   # [6, 15, -4]

Keeping Dimensions with keepdims=True

By default, axis operations return Rank 1 arrays. If you need to maintain 2D shape (important for matrix multiplication in ML), use keepdims=True.

Without keepdims

a.sum(axis=1) # [6, 15, -4]

Shape: (3,)

With keepdims=True

a.sum(axis=1, keepdims=True) # [[6], [15], [-4]]

Shape: (3, 1)

1.8 — Reshaping & Transpose

Reshaping Arrays

You can change the shape of an array without changing its data. The key rule: the total number of elements must stay the same.

The Reshape Rule

Original elements = New rows × New columns

10 elements can become: 2×5, 5×2, 1×10, 10×1
10 elements cannot become: 3×3 (that's only 9 spots!)

my_array = np.arange(10)  # [0, 1, 2, ..., 9], shape (10,)
✓ my_array.reshape(2, 5) 2 × 5 = 10
✓ my_array.reshape(5, 2) 5 × 2 = 10
✓ my_array.reshape(1, 10) 1 × 10 = 10
✗ my_array.reshape(3, 3) 3 × 3 = 9 ≠ 10 — ERROR!

Pro Tip — Chain arange() and reshape()

np.arange(20).reshape(4, 5) — Creates numbers 0–19 arranged in a 4×5 matrix. Very useful for quickly building test arrays.

Transpose: Flipping Rows and Columns

Transpose flips an array over its diagonal — rows become columns and columns become rows.

Does Transpose Change the Original?

No! A.T gives you a view of A transposed, but A itself stays the same. If you want to save the transpose: B = A.T

Exam-Style Question

You might see: "Calculate AT + B - A"

Steps: (1) Transpose A first, (2) Add B, (3) Subtract A.
Remember to check shape compatibility at each step!

Key Takeaways

✓Aggregation reduces arrays to summary values: sum, mean, max, min, std.
!Use np.sum() not Python's sum() — it's faster and correct for arrays.
✓axis=0 goes DOWN (columns) · axis=1 goes ACROSS (rows).
✓keepdims=True maintains 2D shape after aggregation.
✓Reshape: Total elements must stay the same (rows × cols = original count).
✓A.T transposes (flips rows ↔ columns) without changing the original.

Code Cheat Sheet

# Aggregation Functions
a.sum()                        # Sum all elements
a.mean(), a.max(), a.min()     # Other aggregations
np.sum(a)                      # Function syntax (same result)

# Axis Operations
a.sum(axis=0)                  # Sum columns (go DOWN)
a.sum(axis=1)                  # Sum rows (go ACROSS)
a.sum(axis=1, keepdims=True)   # Keep 2D shape

# Reshaping
np.arange(10)                  # [0, 1, 2, ..., 9]
my_array.reshape(2, 5)         # Reshape to 2 rows, 5 cols
np.arange(20).reshape(4, 5)    # Chain: create + reshape

# Transpose
A.T                            # Transpose (rows ↔ columns)
B = A.T                        # Save transposed version
