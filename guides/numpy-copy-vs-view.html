Copy vs View

Lecture 6 Dr. Mahyar Sharif Vaghefi UT Arlington

This Catches People Off Guard Constantly

In regular Python lists, when you slice, you get a copy.

In NumPy, when you slice, you get a view — it's still pointing to the same memory!

The Problem: Views Share Memory

              [ 8,  9, 10],

Why Does This Happen?

You changed B, but A changed too! They're sharing memory. NumPy doesn't want to waste memory by copying data unnecessarily. So B doesn't get its own copy — it just points to the same memory location as A.

B doesn't get its own copy! It's just a "window" into A's memory.
When you change B, you're changing the underlying memory that A also uses.

The Solution: Use .copy()

Use .copy() to Create an Independent Copy

B = A[1:, 1:].copy() — B is now independent!
B[1, 1] = 100 — Only B changes, A stays the same.

View vs Copy Comparison

View (Default)Copy (.copy())
Syntax B = A[1:, 1:] B = A[1:, 1:].copy()
Memory Shares memory with A Own separate memory
Change B A changes too! Only B changes
Speed Faster (no copying) Slower (copies data)
Memory Use Efficient (no extra memory) Uses more memory

Real-World Implications — This Can Destroy Your Data

Imagine: (1) You load a large dataset into original_data. (2) You slice out a portion to experiment with: test_data = original_data[100:200]. (3) You make changes to test_data, thinking you're safe. (4) Your original dataset is now corrupted!

Always use .copy() when you want to modify a slice independently.

Key Takeaways

!NumPy slices create views, not copies (unlike Python lists).
!Views share memory — changing one changes the other!
✓Use .copy() to create an independent copy: B = A[1:, 1:].copy()
✓Views are faster and memory-efficient — use them when you don't need to modify.
✓Copies are safer — use them when you need to modify independently.

Code Cheat Sheet

# DANGEROUS: Creates a view (shares memory)
B = A[1:, 1:]              # B is a VIEW of A

# SAFE: Creates an independent copy
B = A[1:, 1:].copy()       # B is a COPY

# Check if B is a view or copy
B.base is A                # True if view, False/None if copy
np.shares_memory(A, B)     # True if they share memory
